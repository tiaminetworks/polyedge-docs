PolyEdge Orchestrator API (1.1)

Download OpenAPI specification:

The Orchestrator API configures the 5G NR radio and RF processing pipeline, and starts, stops, and monitors the managed radio and ISAC Engine processes. It backs the PolyEdge web GUI directly, so every action available in the GUI is available here for automation and integration.

System

Trigger a self-update

Fetches the configured update manifest and reinstalls PolyEdge's packages. Disabled by default; must be explicitly enabled in the deployment's environment configuration. Intended for the GUI's update control, not routine client automation.

Authorizations:
ApiKeyAuth

Responses

Default local/dev

http://localhost:3000/api/system/update

Response samples

  • 401
  • 403

Write NR radio configuration

Updates the 5G NR radio parameters. Takes effect the next time the radio starts; does not affect an already-running radio session.

Authorizations:
ApiKeyAuth
Request Body schema: application/json
required
operator
string

Network operator label

band
integer

3GPP NR band number

frequency
number

DL carrier center frequency in Hz (100 MHz - 10 GHz)

subcarrier_spacing
number
Enum: 15000 30000 60000

SCS in Hz; validity depends on the selected band

number_of_prbs
integer [ 1 .. 275 ]
integer or Array of integers

Physical cell ID, or a list of candidate PCIs to scan

object (GnbLocation)

The illuminator's (gNB/cell tower) position, used as the TX reference point for coordinate transformations.

object (ReconfigConditions)

Thresholds that determine when the radio should attempt to re-synchronize to a stronger or cleaner cell.

sweep
object

Optional multi-carrier sweep definition, for scanning multiple operator/band combinations in sequence.
